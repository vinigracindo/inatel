Teste Técnico Inatel

1. Criar uma interface gráfica para Login e Tela principal do sistema

--> http://localhost:8000/login/ e http://localhost:8000/

2. Tela de Login contendo os campos “usuário” e “senha” previamente cadastrados

--> http://localhost:8000/login/
--> Usuário criado no momento do deploy com docker (admin/admin).

3. Criar um botão que deverá consumir uma API pública (de sua escolha) e exibir os resultados na tela

--> Menu: Clima (Dados API). URL: http://localhost:8000/weather/
--> API que acessa o a previsão de clima pelos próximos 4 dias passando as coordenadas.

4. Criar um campo na tela para selecionar um arquivo dos tipos “.xlsx” ou “csv”

--> Menu: Importar Dados. URL: http://localhost:8000/core/schedule/import/
--> Realiza Verificação de Tipos. Aceita apenas arquivos CSV e XLSX.
--> O arquivo é lido, verificado e retorna mensagem de erro caso encontre alguma inconsistência.
--> O arquivo inatel.csv na raiz do projeto é um modelo.
--> Os dados do arquivo são salvos nos models do módulo core.

5. Criar uma “view” com “serializers” para exibir dados dos arquivos em uma tela específica

--> Menu: Visualizar Importação. URL: http://localhost:8000/core/schedule/list/
--> É listado todas as importação. Ao clicar na ação "Visualizar dados do arquivo" é mostrado os dados serializados.

6. Criar o Admin Panel para validação dos dados

--> Utilizei e customizei o próprio Admin do Django para visualizar os dados importados.
--> A validação ocorre de forma automática durante o upload do arquivos. Qualquer alteração de dado pode ser realizado no Admin.
--> http://localhost:8000/admin

7. Criar Logs de histórico de tudo que for realizado no sistema

--> middleware localizado em logs/middeware.py
--> É salvo em outro banco de dados como pode ser visto no arquivo .env.example
